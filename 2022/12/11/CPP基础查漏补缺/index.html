换行操作符

std::endl 是一个换行操作符，经常出现在 std::cout 输出的末尾，其作用是输出一个换行符，并强制将stream buffer中的数据刷入到对应的设备中去。例如我们调用一次 std::endl 的时候，会强制将stream buffer中的数据刷到命令行。在程序比较简单的时候，这个刷新buffer的操作不是很明显，因此看起来 std::endl 和 \n 没什么区别。但需要明确的是 std::endl 是一个操作符，而 \n 只是一个转义字符，两者天差地别。

基本内置类型

C++中有基本内置类型，也可以自定义变量或者说对象类型。基本内置类型分为两大类

  1. 算术类型
    1. 整型
      1. 整型 int
      2. 字符型 char
      3. 布尔型 boolean
    2. 浮点型 float
  2. 空类型 void

在C++的认知中，除了空的变量以外，其他的变量都可以看成是数。而计算机能表示的数又只需要分成整数与浮点数，字符、布尔量统统都用整数表示。

字符类型分类

C++中的字符主要可以分成4类

  1. 普通字符 char，在内存中占8位
  2. 宽字符 wchar_t，在内存中占16位
  3. 16位字符 char16_t，在内存中占16位，用来表示2字节Unicode字符
  4. 32位字符 char32_t，在内存中占32位，用来表示4字节Unicode字符

注意，对于3字节的Unicode字符，C++中没有提供对应的类型，直接用 char32_t 存就行了。由于目前的字符集最大也只有32位，因此不存在64位的字符类型。

整型分类

C++中的整型主要可以分为2大类，一类是 unsigned，即无符号整型；另一类是 signed，即有符号整型。

根据长度的不同，整型又可以分为4类，分别是

  1. 短整型 short，一般是16位
  2. 普通整型 int，一般是16位
  3. 长整型 long，一般是32位
  4. 超长整型 long long，一般是64位

在不同的处理器上，C++的上述几种整型对应的字节长度可能是不一致的。例如嵌入式领域中，处理器一般是32位的，而服务器或家用电脑的处理器则普遍是64位的，所以在开发的时候需要特别注意整型的字节长度变化。

注意，有符号整型的类型转换可能会出现 undefined 情况，这种情况发生在一个有符号整型被赋予了超出其表示范围的数。

基本内置类型的字面值常量

字面值常量，即 literal，指的是代码中出现的形如 'a'，123 这样的常量。我们平时可能不太注意字面值常量的类型，但事实上字面值也有不同的类型，最明显的就是 12 是整型的字面值常量，而 1.2 则是浮点型的字面值常量。

字面值常量的类型可以由前缀、后缀以及字面值共同判断，下面给出对应于不同字面值常量类型的标识

  1. 整型 - 用后缀标识不同类型
    1. u 或 U 表示 unsigned，不带 u 或者 U 默认为 signed
    2. l 或 L 表示 long 类型，ll 或 LL 表示 long long 类型，不带默认为 int 类型
    3. 上述两种后缀可以结合使用，例如 123ULL 表示 long long 类型的无符号整型
  2. 浮点型 - 用后缀标识不同类型
    1. f 或 F 表示 float 类型
    2. l 或 L 表示 double 类型
    3. 例如 0.168F 表示32位浮点数，3.14L 表示64位浮点数
  3. 字符型/字符串 - 用前缀标识不同类型
    1. u 表示 char16_t 类型
    2. U 表示 char32_t 类型
    3. L 表示 wchar_t 类型
    4. u8 表示 utf-8 类型字符串

一般很少在代码中出现特别需要标识的字符型字面值，因为一般来说代码都是用ASCII字符集内的字符写成的。

还有一个C++11标准引入的新特性，即指针的字面值 nullptr。在C++11以前，如果要定义一个空指针，往往通过例如 int *p = 0; 的语句来定义，但是C++11以后，int *p = nullptr; 这种方式定义一个空指针更被推荐。

默认初始化

基本内置类型的变量如果在全局作用域被定义，但没有显式初始化，那么编译器会默认将其初始化为0；若在非全局作用域被定义，且没有显式初始化，那么编译器会认为这个变量的值是 undefined。而自定义类往往会提供一个无参数的构造函数，所以提供了无参数构造函数的自定义类在定义时不进行显式初始化也没有关系，但如果某一个类仅提供有参数构造函数，那么必须进行显式初始化，且必须给相应的参数，否则编译器将会报错。

声明和定义

C++支持分离式编译(separate compilation)，这使得同一个变量可以在不同的文件中出现，同时也意味着变量的定义和声明应该分离。先来明确变量的定义和声明的区别：

  • 定义(definition)：为变量分配相应的内存
  • 声明(declaration)：令变量在当前作用域有效

也就是说，定义一个变量就是创建一个变量，而声明一个变量，只是让程序知道当前作用域有这样一个可访问的变量。

有没有可能只定义但不声明一个变量呢？比如，在一个头文件中完成对变量的定义（大概率是一个常量），在其他源码中可以通过声明来引用这个变量。这是可以的。前面提到，所有含有显式初始化的语句都是定义，所以显式初始化一个变量，但又这句语句加上 extern 修饰符，这个语句就是一句定义，而不是一句声明了。需要注意，一般这种语句仅仅出现在头文件，或者说出现在全局作用域，因为内容写在头文件和写在全局作用域是等价的；如果块作用域中出现定义但不声明的语句，编译器将会直接报错，因为这没有任何意义。
